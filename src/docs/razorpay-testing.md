# Razorpay Integration & Sandbox Testing

To facilitate transaction development without hitting real billing limits, Emotist provides a pre-configured standalone **Razorpay Test Environment**.

---

## 🏗️ Razorpay Sandbox Architecture

The Razorpay simulator mimics checkout actions, allows you to enter fake credit card details, generates mock payment IDs, and signs checkout parameters matching Razorpay API specs.

```
[Frontend App] ──► Create Order ──► [API Server] ──► Return Order ID
                                                           │
                                                           ▼
[Checkout Webpage] ◄── Simulate Payment ◄── Load URL with order_id
```

---

## 🚀 Running the Simulator

Run the checkout sandbox harness from the monorepo root:

### Option 1: Full Test Environment (Backend API + Frontend Checkout)
This runs both the mock backend API and the web forms together:

```bash
npm run razorpay:test
```
- Open [http://localhost:3500](http://localhost:3500) in your browser.

### Option 2: Frontend Client Only
If you are already running your main NestJS API server locally, you can start only the checkout frontend:

```bash
npm run razorpay:frontend
```
- Open [http://localhost:3501](http://localhost:3501) in your browser.

### Option 3: Quick Checkout Page
A stripped-down form to test specific order payloads directly:

```bash
npm run razorpay:quick
```
- Access the page directly at [http://localhost:3501/checkout.html](http://localhost:3501/checkout.html).

## 🧪 Testing Specific Orders

To bypass the order creator form and test a specific order ID generated by your local API:

1. Copy the `order_id` (e.g. `order_M8n3j4h7c6v5x4`) from your database or console output.
2. Load the checkout page directly with the order ID in the URL query string:

```
http://localhost:3501/checkout.html?order_id=order_M8n3j4h7c6v5x4
```

## 🔐 Verifying Webhook Signatures

Upon successful transaction completion, Razorpay returns:
- `razorpay_payment_id`
- `razorpay_order_id`
- `razorpay_signature`

The backend verifies signatures using the crypto HMAC module:

```javascript
const crypto = require('crypto');

function verifySignature(orderId, paymentId, signature, secret) {
  const text = `${orderId}|${paymentId}`;
  const generatedSignature = crypto
    .createHmac('sha256', secret)
    .update(text)
    .digest('hex');

  return generatedSignature === signature;
}
```

Ensure your `STRIPE_WEBHOOK_SECRET` or Razorpay key corresponds to your local credentials configuration.
